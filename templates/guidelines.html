All packages must follow a set of basic rules:

  • Use the appropriate package repository (see below)
  • Declare dependencies with full names (i.e. "core/configlib" instead of "configlib")
  • Libraries should place include files or headers in include/. These won't be sent to the calculator under normal circumstances, but will be used by the SDK.
  • Libraries must not use a library ID that's already in use by another package.
  • All packages implicitly grant packages.knightos.org a worldwide, non-revokable license to distribute that package.

We'll fix simple errors ourselves and contact you via the email in your profile if we need to run something past you.

Package Repositories

We have a few repositories, each with their own purpose. Use the right one or your package will not be approved, or may be changed prior to approval:

  • core: Core packages from the KnightOS team, essential for normal operation
  • extra: Extra packages from the KnightOS team, non-essential for normal operation
  • community: Packages from the community at large
  • ports: Packages ported from TIOS or other operating systems
